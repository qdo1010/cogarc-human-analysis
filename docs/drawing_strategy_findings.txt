DRAWING-STRATEGY FINDINGS
Experiment-2 edit sequences on the 75 CogARC tasks
=====================================================

Two questions answered:

    1. Which Success-grid components do humans draw first?
    2. When a human pauses and then draws again, what is the
       "cognitive chunk" they just produced, and how does it relate
       to the connected-component segmentation a GNN prior would
       use?

Population:
    14,107 trajectories · 233 subjects · 75 tasks · 1,491 Success
    components · 130,015 chunks.


------------------------------------------------------------------------
1. COMPONENT PRIORITY  (which objects get drawn first)
------------------------------------------------------------------------

Method
------
For every Experiment 2 trajectory we scan the edit sequence in order.
The moment an edit lands on a cell of a given 4-connected Success
component, that component is marked "touched". Its rank is the index
of that edit in the full trajectory, normalized to [0, 1] (0 = first
edit of the trajectory, 1 = last). Averaging across participants gives
a per-component "touch priority" — lower values mean the component is
drawn earlier.

We then regress touch priority on per-component features with task
fixed effects (z-score features and target within each task, then run
simple OLS per feature). This isolates *which attributes of a
component* make it attract early attention, controlling for task.

Results
-------
Across 1,316 components with usable data (1,491 - ones no one touched):

    feature                 beta         Spearman
                         (within-task)     rho    p
    ---------------------------------------------------
    area                    -0.253       -0.146  <.001
    bbox_w                  -0.238       -0.137  <.001
    is_unique_color         -0.231       -0.077   .005
    bbox_h                  -0.220       -0.122  <.001
    color_frac_among_nonbg  -0.112       -0.119  <.001
    dist_center             -0.071       -0.071   .010
    is_rarest_color         -0.055       -0.029   .289
    boundary_frac           +0.230       +0.075   .007
    fill_ratio              +0.272       +0.130  <.001
    bbox_aspect             +0.005       +0.022   .434

Interpretation
    Negative beta = higher values of the feature predict EARLIER touch.
    Positive beta = higher values predict LATER touch.

    The rank order of attention is:

        1. Big components (area, bbox_w, bbox_h)
        2. Components whose color appears nowhere else
           (is_unique_color)
        3. Components in the dominant non-background color
        4. Components closer to the center
        5. Dense, compact shapes
        6. ... while thin / high-boundary / sparse shapes are saved
           for last (boundary_frac and fill_ratio both flip sign).

    "Rarest" in a count sense (the color with the fewest components
    in the task) is not a reliable predictor on its own; it is
    "unique" (exactly one component of that color) that matters.

Actionable takeaway
    A human-informed GNN attention prior should initialize node
    importance using a monotone function of area, color-uniqueness,
    centrality, and compactness. These are cheap to compute from the
    input grid before any message passing.


------------------------------------------------------------------------
2. DRAWING CHUNKS  (what is the participant's cognitive unit?)
------------------------------------------------------------------------

Method
------
Each trajectory is split at pauses longer than

        max( 2 * subject_median_rt ,  500 ms ),

yielding a sequence of chunks. Within each chunk we record:

    size                   — number of edits in the chunk
    n_cells                — distinct cells edited
    n_colors               — distinct colors placed
    color_homogeneity      — fraction of edits using the modal color
    is_connected           — whether the cells form one 4-connected blob
    cc_count               — number of 4-connected components of the cells
    bbox / bbox_area       — spatial footprint

For each chunk we also compute, against the task's Success grid:

    success_iou_best              IoU vs the best-matching Success CC
    success_iou_color_class       IoU vs all Success cells of one color
    n_success_cc_spanned          how many distinct Success CCs the
                                  chunk touches (0 if the chunk is
                                  entirely on wrong cells)

Results
-------
Chunks are small and nearly monochromatic:

    median chunk size .......... 3 edits
    median color homogeneity ... 1.00   (i.e. single-color)
    median n_colors ............ 1
    mean chunks per trajectory . 9.2

Composition across the 130,015 chunks:

    41 %   entirely outside Success  (all edits on wrong cells)
    37 %   inside exactly one Success connected component
    21 %   span 2 or more Success connected components,
           single-color         <-- "color-over-connectivity" chunks
     1 %   span 2 or more Success components, mixed color

The headline number: within the 22 % of chunks that cross CC
boundaries, 95 % stay in one color. So when people violate the
connected-component segmentation, they do so by grouping disjoint
SAME-color regions — not by jumping between different colors.

Per-task variation is large. Tasks where the solution rule itself is
color-class (like "paint every square of color C") induce very high
color-over-CC rates:

    task_id    frac_chunks_multi_cc
    -------    --------------------
    0d3d703e   94 %
    3aa6fb7a   89 %
    6ea4a07e   73 %
    0b17323b   72 %
    bcb3040b   68 %
    32597951   58 %
    d511f180   62 %

On tasks where the solution is "complete one shape, then the next",
the multi-CC rate drops below 20 % and chunks align cleanly with
components. So the relative prominence of the two segmentation modes
is itself a per-task property of the solution rule.

Interpretation
    Humans do chunk, and their chunks are small monochromatic units.
    Most chunks stay inside one connected component, consistent with
    object-at-a-time drawing. But a substantial and systematic
    minority of chunks — roughly one in five — cross CC boundaries
    while holding color constant, which means the primary
    segmentation axis that drives chunking is COLOR. Connectivity is
    secondary. On tasks whose solution rule is color-indexed, the
    ordering reverses and color becomes dominant.

Actionable takeaway
    A single connected-component prior (hierarchical, containment,
    or 4-connected segmentation) loses roughly one in five cognitive
    chunks. A color-class prior is no better on its own because it
    over-merges components of the same color into one giant node.
    The prior that matches human behaviour is a HYBRID: nodes indexed
    by (color, component_id) with an additional edge family linking
    all components of the same color. The per-task fraction
    frac_chunks_multi_cc can be used directly to weight how much the
    color edges dominate over the connectivity edges in the graph.


Secondary segmentation axes (what else is a chunk beyond color?)
----------------------------------------------------------------
Color dominates within-chunk organization by construction — median
color homogeneity is 1.0. The more informative question is:
given that a chunk is a single color, what SPATIAL pattern does it
follow?

Restricting to the 87,201 color-homogeneous chunks of size >= 2
(67 % of the full 130 k; size-1 chunks are atomic and have no
secondary structure):

    Organizing axis                                              rate
    ----------------------------------------------------------   ----
    one 4-connected blob                                         69 %
    all cells on a single row                                    24 %
    all cells on a single column                                 21 %
    all cells on one diagonal                                     6 %
    pen adjacency   (fraction of consecutive edits
                      at Chebyshev distance <= 1)          mean  0.85
    bbox fill ratio (n_cells / bbox_area)                  med   0.79

Mutually exclusive buckets (adding to ~100 %):

    connected 2D blob (not row / col)                            33 %
    connected horizontal line    (same_row AND connected)        20 %
    connected vertical line      (same_col AND connected)        17 %
    connected diagonal line                                       6 %
    same-row only (disconnected horizontal jumps)                 4 %
    same-col only (disconnected vertical jumps)                   4 %
    disconnected, off-axis (the jumping chunks)                  17 %

Interpretation
    The hierarchy of drawing axes, ordered by how strongly they
    organize a chunk:

        1. COLOR              — 95 %+ of chunks are monochromatic.
        2. SPATIAL CONTIGUITY — ~70 % of chunks are a single
                                 4-connected blob, and the pen
                                 moves to a touching cell ~85 % of
                                 the time.
        3. AXIS ALIGNMENT     — 37 % of chunks are axis-aligned
                                 line segments (horizontal or
                                 vertical). "Draw a horizontal bar"
                                 and "draw a vertical bar" are two
                                 of the most common micro-strategies.
        4. DIAGONAL           — rare but not zero, 6 %.

    The 17 % "disconnected, off-axis" slice is the color-indexed
    strategy: chunks where only color is shared between the cells,
    and the participant is grouping same-color regions across
    disjoint parts of the grid.

Actionable takeaway
    After color, spatial connectivity is the second-most-important
    segmentation axis, but axis-alignment (pure row / pure column)
    explains a further 37 % of chunks and is currently absent from
    the ARC-GNN priors. Adding a pair of "row" and "column"
    abstractions to the prior bank would directly model the
    "draw one line at a time" micro-strategy that humans use roughly
    as often as they fill in 2D blobs.

3. ARE HUMANS JUST FOLLOWING THE TASK RULE?  (strategy comparison)
------------------------------------------------------------------------

Method
------
A natural objection to Section 2 is that if a task's Success grid is
itself a collection of horizontal bars, chunks will trivially look
horizontal — there is nothing else to chunk. To rule this out we
generated six synthetic "drawing strategies" that each produce the
same Success grid but with different chunk organization, then
compared each strategy's chunk fingerprint against the real humans.

Strategies:

    random_k3        cells in random order, chunks of size 3
    color_first      paint all of color A, then color B, ...;
                     one chunk per color (no intra-color order)
    nn_color_first   color_first, but within each color paint in
                     nearest-neighbor order starting top-left
    object_first     one 4-connected component per chunk, painted
                     in nearest-neighbor order
    row_first        one row of non-background cells per chunk
    col_first        one column of non-background cells per chunk

For each strategy on each task we recompute the same ten chunk-level
features (mean chunk size, frac_single_color, frac_connected,
frac_same_row, frac_same_col, frac_multi_cc_spanned,
frac_multi_cc_same_color, frac_one_cc, mean_iou_cc,
mean_iou_color_class) and compare with the humans' per-task feature
vector using column-z-scored Euclidean distance.

Results
-------
Per-task closest-strategy counts:

    object_first    25 / 75
    random_k3       17 / 75
    row_first       14 / 75
    col_first       11 / 75
    color_first      8 / 75
    nn_color_first   0 / 75

Mean distance (lower = closer to humans):

    object_first    3.48
    row_first       3.68
    col_first       3.68
    random_k3       3.72
    color_first     3.84
    nn_color_first  3.84

Interpretation
    Humans are NOT any single canonical strategy. One third of tasks
    are best explained by an object-first strategy, but 40 % of tasks
    are closest to row/col/color strategies, and nearly a quarter are
    closest to a random baseline. If participants were simply
    bottlenecked by the task rule (only one valid way to chunk), one
    strategy would win on almost every task. It does not.

    This matters because it says humans have genuine behavioral
    flexibility: the chunking pattern shifts per task, and on any
    given task humans do look more like one canonical strategy than
    another. Rank-1 strategy differs across tasks, so the aggregate
    chunking fingerprint is a blend.

Actionable takeaway
    A single prior class (whether connected-component or color-class
    or row/col) will only fit a subset of tasks. A GNN that picks
    its prior per task using a cheap behavioral predictor is
    strictly better justified than one with a fixed prior, because
    humans themselves switch per task.

Do task groups (Obj / Geo / Num / GoD or complexity) share a strategy?
----------------------------------------------------------------------
Merging the per-task winner table with the CogARC Task-tags CSV, no
task group exhibits a strongly dominant strategy. Every primary tag
has a plurality winner but each plurality is at best 44 %, not a
tight cluster.

    Winner counts by primary tag
        tag  n   object  random  row  col  color  (plurality)
        Obj  19       7       2    3    4      3   object_first 37 %
        Geo  30       9      11    5    3      2   random_k3    37 %
        Num  14       3       3    5    1      2   row_first    36 %
        GoD   9       4       1    0    3      1   object_first 44 %

    Winner counts by complexity
        cx   n   object  random  row  col  color  (plurality)
        1    20       7       6    5    1      1   object_first 35 %
        2    37       9      10    5    8      5   mixed
        3    15       7       1    3    2      2   object_first 47 %

Complexity matters more than tag. Hard tasks (complexity 3) shift
sharply toward object_first (47 % vs 35 % overall). On hard tasks
humans invest in object-level reasoning; on easy / medium tasks they
are more flexible.

Dominance gap. For each task we computed the distance of its best
strategy vs its second-best. The median gap is 0.25 out of baseline
distances around 3.5, meaning on half the tasks at least two
strategies fit humans within ~7 % of each other. Strategies are not
only heterogeneous across tasks, they are also ambiguous within
tasks.

Bottom line: there is no canonical "way of drawing a task" that is
stable across a group. Human drawing is task-level flexible. The
only robust group-level effect is that harder tasks push behavior
toward object-level chunking.

4b. WHAT ABOUT THE TASK PREDICTS WHICH STRATEGY HUMANS USE?
------------------------------------------------------------------------

Given that humans match different canonical strategies on different
tasks (Section 4), is something about the TASK itself forcing the
choice? We separated two hypotheses WITHOUT using the Obj/Geo/Num/GoD
tags:

  (H1) The task affords only one plausible strategy, so every human
       picks it because there is nothing else to pick.
  (H2) The task affords several strategies and humans converge on one
       of them because the task features bias the choice.

Operationalisations:

  * affordance(task, s) = True iff the synthetic strategy produces
    >= 2 chunks on the task's Success grid.
  * n_afforded(task)    = how many of the 6 canonical strategies the
                          task affords (0..6).
  * human_entropy(task) = Shannon entropy (bits) over the distribution
                          of each subject's best-matching strategy for
                          that task.
  * dominant_frac(task) = fraction of subjects choosing the modal
                          strategy on that task.

Pipeline: human_task_vs_strategy.py
Outputs:  prior_analysis/task_features_for_strategy.csv
          prior_analysis/subject_best_strategy.csv          (12 508 rows)
          prior_analysis/task_vs_strategy_correlations.csv
          prior_analysis/task_vs_strategy_usage_correlations.csv

Findings:

  (a) Affordance is nearly constant across tasks. 67 of 75 tasks
      afford ALL 6 canonical strategies; the remaining 8 afford 3-4
      (they are all single-output-color tasks that disable
      color_first / nn_color_first). So H1 does almost no work: the
      task hardly ever rules strategies out.

  (b) Humans do NOT fill all afforded strategies. Mean
      dominant_frac = 0.68 (std 0.19); on 10+ tasks one strategy
      collects >= 90 % of subjects. Tasks range from near-perfect
      agreement (dominant_frac > 0.95, entropy < 0.5 bits) to near-
      chance (dominant_frac ~0.28, entropy > 2.0 bits).

  (c) Aggregate predictors of entropy / dominant_frac are WEAK.
      Spearman |rho| < 0.16 for n_afforded, grid size, color counts,
      symmetry, and interface-shortcut rates (used_copy, n_showex,
      n_reset, n_edits). No single feature on its own predicts
      whether subjects will converge.

  (d) Per-strategy usage IS predicted by task features, and the
      predictors are interpretable. For each canonical strategy we
      correlate the subjects' per-task usage share with task features:

         Strategy        Predictor          Spearman ρ   p
         ------------    ---------------    ----------   -------
         object_first    n_success_cc       +0.39        0.001
         color_first     n_colors_output    +0.48        <0.001
         color_first     n_new_colors       +0.25        0.028
         col_first       n_reset_mean       +0.34        0.003
         col_first       symmetry_h         +0.30        0.010
         row_first       n_success_cc       -0.27        0.021
         random_k3       mean_cc_area       -0.58        <0.001
         random_k3       max_cc_area        -0.47        <0.001
         random_k3       n_cells_nonbg      -0.31        0.007

      Reading this:
        - More distinct connected components in Success => more
          subjects draw object-by-object.
        - More output colors => more subjects sort by color.
        - Small outputs (few cells, tiny objects) produce chunk
          patterns that match the synthetic random_k3 baseline; this
          is mostly a measurement artefact (small outputs leave
          little room for any single strategy signal to emerge) and
          should be interpreted as "no canonical strategy dominates"
          rather than "subjects really randomise".
        - Horizontal symmetry and higher n_reset favour column-by-
          column drawing; the reset signal is consistent with
          subjects iterating vertical sweeps, hitting a mistake,
          resetting, and restarting a column.

  (e) The interface shortcuts (used_copy_mean, n_showex_mean,
      n_reset_mean, n_edits_mean) show up only weakly. The strongest
      interface signal is n_reset_mean -> col_first (+0.34); copy
      usage and show-example usage do not meaningfully shift the
      chosen strategy.

Takeaway. Affordance is almost always 6; the task does NOT force
humans into a single strategy. But task-intrinsic structure
(#connected components, #colors, object size) biases WHICH strategy
subjects most often match, and that bias is strong enough to give
object / color / random-like signatures distinguishable at Spearman
rho = 0.4-0.6. Humans reconcile task structure with personal style,
and the structure-side of that reconciliation is what the heat-map
in task_vs_strategy_figure.png visualises.

Figure: prior_analysis/task_vs_strategy_figure.png  (panels A-E
described in the figure docstring).

5. CAVEATS
------------------------------------------------------------------------

- The 41 % "outside Success" chunks are partly participants who are
  drawing a coherent but WRONG answer (the Top Error patterns). They
  still look like chunks but they are not aligned with Success by
  construction.

- Touch priority is normalized by trajectory length; a component that
  is the last thing in a 5-edit trajectory and a component that is
  the last thing in a 200-edit trajectory both score 1.0. This is
  intentional (priority is about relative ordering within a
  participant's plan) but means the metric is insensitive to how
  LONG participants spend before reaching a component.

- The chunking pause threshold ( max(2*median_rt, 500 ms) ) is
  defensible but tunable. Loosening it to 1.5x or tightening to 3x
  shifts mean chunk size by 1-2 edits; the rank ordering of tasks
  by frac_multi_cc is robust (Spearman > 0.9 across reasonable
  threshold choices).

- Experiment 2 trial order is randomized per subject and is NOT
  exposed in the Zenodo release. All analyses here pool over the
  entire session; within-subject progression effects can only be
  studied on Experiment 1 (fixed task order, 49 subjects).
